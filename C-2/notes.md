### Understanding Angular

#### What is Angular?

- It is a JavaScript framework for building web applications.
- It is a client-side framework that uses TypeScript as its primary language.
- It is a component-based framework that uses HTML, CSS, and JavaScript as its building blocks.
- It was created by Google in 2010 and is now maintained by a community of developers.

#### Why do we need a framework?

- It provides a structure for building web applications.
- It provides a set of tools and libraries that make it easier to build web applications.
- It provides a set of best practices for building web applications.

#### Benefits of using Angular

- It is a Typescript based framework.
- It has a powerful Command Line Interface (CLI) tool that makes it easy to create and manage projects.
- It has a rich ecosystem of libraries and tools that can be used to build web applications.
- It has two-way data binding, which makes it easier to build dynamic web applications.
- It uses Dpendency Injection, which makes it easier to manage dependencies between components.

#### SPA vs Traditional Web Apps

- Single Page Applications (SPAs) are web applications that load a single HTML page upon the initial visit. After this initial load, content updates and navigation within the application are handled dynamically by JavaScript on the client-side, typically by fetching data (e.g., JSON) from the server and updating parts of the existing page (the DOM) without requiring a full page reload. This provides a more fluid, app-like user experience.

- Traditional Web Apps (also known as Multi-Page Applications - MPAs) are web applications where each user interaction 
(e.g., clicking a link, submitting a form) typically triggers a request to the server, which then renders and sends back an entirely new HTML page to the browser. This results in a full page reload for most navigations, meaning the browser effectively "leaves" the current page and "loads" a new one from the server.
